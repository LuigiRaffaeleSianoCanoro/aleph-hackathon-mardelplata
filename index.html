HERRAMIENTAS
PARA TRABAJAR

No hace falta instalarlas todas. Elegí las que más se adapten a lo que vas a construir. La mayoría son gratuitas o tienen un plan free suficiente para el evento.

⚙ Construcción y código
v0
Freemium Web

Generador de interfaces web con IA de Vercel. Describís lo que querés en texto y genera código React listo para usar. Perfecto para armar el frontend de tu proyecto rápido sin saber programar en detalle. Podés iterar con prompts en segundos.

→ v0.dev
Visual Studio Code
Gratis Win / Mac / Linux

El editor de código más usado del mundo. Donde vas a escribir y editar todo tu código durante la hackathon. Tiene miles de extensiones, soporte para todos los lenguajes y funciona muy bien con GitHub Copilot. Si no lo tenés, instalalo antes del evento.

→ code.visualstudio.com
GitHub Copilot
Free para estudiantes

Asistente de IA dentro de VS Code que completa tu código mientras escribís. Sugiere funciones enteras, corrige errores y explica código. Si tenés cuenta de estudiante en GitHub, el plan Pro es gratis. En la hackathon acelera muchísimo el desarrollo.

→ github.com/features/copilot
Cursor
Freemium Win / Mac / Linux

Editor de código construido sobre VS Code pero con IA integrada mucho más potente que Copilot. Podés pedirle que escriba funciones completas, refactorice código o explique qué hace un archivo. Muy popular entre vibe coders y builders en hackathons.

→ cursor.com
🧠 Inteligencia Artificial
Claude
Freemium Web / App

Asistente de IA de Anthropic. Muy bueno para razonar, escribir código, redactar textos y pensar la arquitectura de un proyecto. Ideal para diseñadores, PMs y personas no técnicas que quieren construir sin programar. También sirve para preparar el pitch.

→ claude.ai
ChatGPT
Freemium Web / App

El asistente de IA más conocido, de OpenAI. Útil para generar código, responder preguntas técnicas, ayudar a pensar ideas de producto, armar documentación y practicar el pitch. El plan gratuito es más que suficiente para la hackathon.

→ chat.openai.com
🎨 Diseño y prototipado
Figma
Gratis Web / App

La herramienta estándar de diseño de interfaces. Podés crear mockups, wireframes y prototipos interactivos del producto antes de programarlo. También sirve para armar slides del pitch. El plan gratuito alcanza para todo el evento.

→ figma.com
Canva
Freemium Web / App

Para quien no maneja diseño profesional, Canva permite armar slides de pitch, presentaciones y materiales visuales de forma rápida usando plantillas. Ideal para el momento de presentar el proyecto al jurado.

→ canva.com
📹 Grabación y demo del producto
FocuSee
Solo Mac Freemium

Grabador de pantalla para Mac que resalta automáticamente el cursor, hace zoom en los clics y genera demos del producto con un look muy profesional. Perfecto para mostrar cómo funciona tu app antes del pitch sin editar el video.

→ focusee.app
OBS Studio
Gratis Win / Mac / Linux

La herramienta de grabación y streaming más usada del mundo. Grabá tu pantalla, tu cámara o ambas al mismo tiempo para hacer el demo del producto. Más compleja que FocuSee pero muy poderosa. Disponible para todos los sistemas operativos.

→ obsproject.com
Loom
Freemium Web / App

Grabá tu pantalla y tu cara al mismo tiempo con un clic. Genera un link para compartir el video al instante, sin edición. Muy usado para hacer demos rápidos del producto y enviarlos a los jurados o compartirlos en redes después del evento.

→ loom.com
🤝 Colaboración en equipo
GitHub
Gratis Web

Donde los equipos guardan, comparten y colaboran en el código. Todos los cambios quedan registrados y el equipo puede trabajar en paralelo sin pisarse. Además, el repositorio del proyecto es lo que los jurados van a revisar: tenelo ordenado.

→ github.com
Notion
Gratis Web / App

Espacio compartido para organizar las ideas del equipo, documentar el proyecto, asignar tareas y llevar notas durante la hackathon. Sirve también para armar la documentación del proyecto que se presenta al jurado.

→ notion.so
🚀 Deploy y publicación
Vercel
Gratis Web

Publicá tu app o sitio web en internet con un solo clic, conectado a tu repositorio de GitHub. Cada vez que subís código, Vercel actualiza el sitio automáticamente. El link público que genera es el que mostrás en el pitch.

→ vercel.com
Supabase
Gratis Web

Base de datos, autenticación de usuarios y almacenamiento de archivos en la nube, todo en un solo lugar y gratis. Si tu proyecto necesita guardar datos o hacer login de usuarios, Supabase es la opción más rápida de configurar durante una hackathon.

→ supabase.com

REGLAS Y CÓDIGO
DE CONDUCTA

Aleph Hackathon es un evento diseñado para desarrolladores, diseñadores, PMs, founders, criptógrafos y entusiastas de la tecnología que quieren colaborar y desarrollar soluciones innovadoras en torno al blockchain y la descentralización. Queremos que todos los participantes tengan una experiencia positiva y enriquecedora. Por eso pedimos que todos muestren respeto y cortesía durante el evento.

🤝 Código de Conducta
  • Respeto y cortesía: Todos los asistentes deben tratarse con respeto. No se tolerará ningún tipo de acoso, ya sea verbal, físico o virtual, relacionado con género, orientación sexual, raza, religión, apariencia, discapacidad u otras características personales.
  • Entorno seguro: Mantengamos un ambiente seguro y profesional. No se permite el uso de imágenes, actividades o materiales sexualizados. Esto aplica a todos: participantes, jurado, mentores, voluntarios y organizadores.
  • Comportamiento responsable: Si alguien te pide que detengas un comportamiento inapropiado, se espera que lo hagas de inmediato. El incumplimiento de estas reglas puede resultar en la expulsión del evento.
  • Reportar incidentes: Si te sentís incómodo/a o creés que alguien está violando el código de conducta, avisanos de inmediato. Podés hablar con cualquier organizador del evento o escribirnos a [email protected].
📌 Reglas del Evento
  • Uso de materiales: Durante la hackathon, las ideas se comparten libremente. Si compartís información que querés proteger, es tu responsabilidad establecer medidas de confidencialidad. Los desarrollos que crees durante el evento son tuyos, pero si usás APIs o software de los sponsors, debés cumplir con sus términos y licencias.
  • Trabajo previo: No está permitido presentar proyectos preexistentes. El trabajo debe comenzar oficialmente cuando arranque la hackathon. De lo contrario, quedás descalificado/a (sin excepciones — todo será verificado). Las funcionalidades de startups existentes están permitidas, pero aplica la regla de código nuevo: el código debe crearse desde cero durante el evento.
  • Grabaciones y fotos: El evento será grabado y fotografiado. Al asistir, aceptás que podemos usar tu imagen y voz en materiales promocionales. Si preferís no aparecer, avisanos para respetar tu privacidad.
  • Condiciones generales: La participación en Aleph Hackathon es bajo tu propia responsabilidad. No nos hacemos responsables por lesiones, pérdidas o daños. Al participar, aceptás estas reglas y condiciones.
